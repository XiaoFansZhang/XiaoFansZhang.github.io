hexo个人博客搭建

发表于 2015-03-17

杂序

这几天和同事讨论关于建博客的话题。我之前的博客是在博客园直接注册申请的，看他们有些是用Hexo配合Github搭建的。
心痒难耐，就去网上查了些资料，瞬间高大上的感觉有木有。以下是一些经验：

准备工作(环境)

Hexo是基于Node.js的静态站点生成框架，所以要运行Hexo就必须安装Node环境。因为本人是通过Git版本工具上传到Github上的，所以也要安装Git的运行环境（Node和Git安装，过几天传上来）。
另外Hexo是使用 Markdown 编写文章的，除了 Markdown 本身的语法之外，还可以使用 Hexo 提供的 标签插件 来快速的插入特定形式的内容。

Hexo

  1. 安装Hexo
    在你的cmd控制台运行如下代码

  2. 创建一个博客文件夹
    为避免不必要的麻烦，路径最好是英文路径，在文件夹里面先右键单击GitBashHere，弹出Git运行窗口（以下所有命令操作都在Git窗口运行），然后执行hexo init。

  3. 测试
    初始化完成后，在文件夹中直接执行 hexo s 在本地测试网页效果，在网页输入这个链接即可：http://localhost:4000/
    正常的话会显示一个带有 Hello World 的文章，此时你的博客在本地就搭建好了

新建文章

hexo new “文章名称”
这句命令执行完毕之后，会在source/_posts中生成以文章名为文件名的markdown文件。
当然直接在这个文件夹里面新建md文件也可以

生成html

hexo g（这一步很关键）
执行这条命令之后，会在public目录下生成所有的静态页面，只要将该目录下所有的文件进行发布，发布到Github上，就可以通过Github的静态页面托管服务来访问页面了。

  1. 提交到版本库
    切换到public目录执行 git init 并进行一次提交

  2. 按照正常流程创建博客，编辑博客

  3. 执行hexo g

总结

对于不熟悉 Git 和 Hexo 基本操作的小伙伴，第一次建博可能比较麻烦。我前面几次安装hexo环境的时候，最后的两个文件等了好久才安装完。一定要等到窗口跳出：Admin@malong MINGW64 /d/myblog (master) 这一行才是执行完成，否者后续的操作就不能继续了。以上有执行过程任何问题，欢迎联系
